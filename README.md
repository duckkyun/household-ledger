# 가계부 (Household Ledger)

설치·서버 없이 **HTML 파일 하나로 동작하는 가계부**입니다. 매일 지출/수입을 기록하고 분류별·월별 통계를 확인할 수 있어요. 데이터는 브라우저(localStorage)에 저장됩니다.

## 사용법

- **로컬**: `index.html` 파일을 더블클릭해서 브라우저로 열면 끝.
- **웹/폰**: GitHub Pages를 켜면 `https://duckkyun.github.io/household-ledger/` 로 어디서든 접속 가능.
  - GitHub 저장소 → **Settings → Pages → Branch: `main` / `/root`** 선택 후 저장.

## 기능

- 지출/수입 기록 (날짜·금액·분류·메모)
- 이번 달 수입·지출·잔액 요약, 월 이동
- 분류별 지출 통계(막대), 최근 6개월 추이 차트
- **JSON 백업/복원**, **CSV 내보내기** (엑셀에서 열림)

## 데이터 보관

기록은 기본적으로 사용하는 **브라우저에만** 저장됩니다. 기기·브라우저를 바꾸면 보이지 않으니, 가끔 **JSON 백업**을 받아두세요. 다른 기기에서는 그 파일을 **JSON 복원**으로 불러오면 됩니다. 여러 기기에서 **자동 동기화**하려면 아래 클라우드 동기화를 설정하세요.

## ☁️ 클라우드 동기화 설정 (Firebase)

구글 로그인으로 폰·PC에서 **같은 기록을 자동 동기화**합니다. 무료 플랜(Spark)으로 충분하고 신용카드도 필요 없어요. 설정은 한 번만 하면 됩니다.

> 참고: `index.html`에 넣는 `firebaseConfig` 값(apiKey 등)은 **비밀이 아니며 공개 저장소에 올려도 안전**합니다. 실제 보안은 아래 Firestore 보안 규칙이 담당해요.

1. [Firebase 콘솔](https://console.firebase.google.com) → **프로젝트 만들기** (이름 아무거나, 애널리틱스는 꺼도 됨)
2. **빌드 → Authentication → 시작하기 → Sign-in method → Google 사용 설정 → 저장**
3. **Authentication → Settings → 승인된 도메인(Authorized domains)** 에 `duckkyun.github.io` 추가 (`localhost`는 기본 포함)
4. **빌드 → Firestore Database → 데이터베이스 만들기** → 위치는 `asia-northeast3`(서울) 등 → **프로덕션 모드**로 시작
5. Firestore **규칙(Rules)** 탭에 아래를 붙여넣고 **게시(Publish)**:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /ledgers/{uid} {
         allow read, write: if request.auth != null && request.auth.uid == uid;
       }
     }
   }
   ```
   → 각 사용자는 **자기 uid 문서만** 읽고 쓸 수 있어, 남의 기록엔 접근 불가.
6. **프로젝트 설정(⚙️) → 일반 → 내 앱 → 웹앱 추가(`</>`)** → 앱 등록 → 나오는 `firebaseConfig` 복사
7. `index.html` 안의 `firebaseConfig`(하단 `<script type="module">`)에 `apiKey / authDomain / projectId / appId` 값을 붙여넣고 커밋 → 끝

설정 전에는 동기화 패널이 "설정 필요"로 표시되고 앱은 **로컬(localStorage) 전용**으로 정상 동작합니다.

### 동작 방식

- **로그아웃** = localStorage에만 저장(기존과 동일).
- **로그인** = Firestore 문서 `ledgers/{uid}` 에 전체 기록을 저장하고 `onSnapshot`으로 실시간 동기화. 오프라인 쓰기는 Firestore 영속 캐시에 큐잉됐다가 온라인 시 반영.
- **첫 로그인** 시 이 기기의 로컬 기록과 클라우드 기록을 `id` 기준으로 병합(이 기기에만 있는 기록이 있으면 합칠지 물어봄).

## 파일

- `index.html` — 앱 전체. 통계·UI는 외부 의존성 없이 동작하고, 클라우드 동기화만 Firebase SDK(CDN, ESM 모듈)를 사용.

## 개발 기록 (시행착오)

만들면서 부딪힌 문제와 내린 결정들을 정리해 둡니다. 나중에 비슷한 걸 만들 때 참고용.

### 1. "가계도"가 아니라 "가계부"

처음 요청이 "가계**도**"였는데 설명은 지출 기록이라 헷갈렸음. 가족 관계도(족보)가 아니라 **가계부**(household ledger)가 맞다고 확인하고 진행. → 애매하면 먼저 확인하는 게 낫다.

### 2. 기술 선택: 단일 HTML + localStorage

서버/DB/빌드 없이 **HTML 파일 하나 + 브라우저 localStorage**로 결정.
- 장점: 파일만 열면 동작, GitHub 관리 간단, Pages로 폰에서도 접속.
- 단점: 데이터가 **브라우저에 종속**됨(기기·브라우저 바꾸면 안 보임). → 이걸 보완하려고 **JSON 백업/복원·CSV 내보내기**를 처음부터 넣음.
- 외부 CDN(차트 라이브러리 등) 대신 **SVG로 직접** 막대·추이 차트를 그려서 오프라인에서도 완전 동작하게 함.

### 3. 저장소 위치

기존 `duckkyun.github.io` 안에 넣을지, 새 저장소로 뺄지 고민 → **새 전용 저장소(`household-ledger`)**로 결정해 깔끔하게 분리.

### 4. 비공개 저장소 + GitHub Pages 함정

저장소를 **private**로 만들었는데, 무료 플랜은 **비공개 저장소에서 Pages 웹 호스팅이 안 됨**. 폰에서 웹으로 쓰려면 저장소를 **공개로 전환**하거나 Pro가 필요. (로컬로만 쓰면 상관없음.)

### 5. 브라우저 테스트: `file://` 차단

자동 테스트(Playwright)로 열려니 `file://` 프로토콜이 **보안상 차단**됨. → `python3 -m http.server`로 로컬 서버를 띄우고 `http://localhost`로 접속해서 테스트하는 방식으로 해결.

### 6. 분류 다중 선택 + "쏘" 추가

- 분류를 **여러 개 중복 선택**할 수 있게 바꿈(칩 토글 → `cats` 배열로 저장). 데이트 지출/수입용 **"쏘" ❤️** 카테고리를 지출·수입 양쪽에 추가.
- **설계 판단**: 한 기록에 분류가 여러 개면 통계에서 금액을 어떻게 셀지가 문제. → 각 분류에 **전체 금액을 중복 집계**(예: 데이트 저녁 ₩35,000이 `식비`·`쏘` 양쪽에 모두 잡힘)하되, **월 합계는 기록당 1회만** 더해 실제 지출과 일치시킴. 그래서 분류별 비율 합은 100%를 넘을 수 있음(정상).
- 예전에 저장한 단일 분류(`cat`) 데이터도 불러올 때 자동으로 `cats` 배열로 변환해 **하위 호환** 유지.

### 7. 날짜칸 잘림 + 금액 콤마

- 날짜 입력칸에 **'일'이 안 보이는 문제**: 값(`2026-07-04`)엔 일자가 있었지만 칸 폭이 `130px`로 좁아 `2026. 07. 04.`의 뒷부분이 **시각적으로 잘림**. → 칸을 넓혀(`1fr 1fr`) 해결. "값이 없다"가 아니라 "안 보인다"였던 케이스.
- **금액에 천 단위 콤마**(`1,100`): `<input type="number">`는 콤마 표시가 불가능. → `type="text"`+`inputmode="numeric"`으로 바꾸고, 입력할 때마다 숫자만 뽑아 `toLocaleString`으로 콤마를 붙임. 저장·계산 시엔 콤마를 제거해 숫자로 파싱.

### 8. 라이트/다크 수동 토글 + 기록 수정

- **테마 토글**: 원래 `prefers-color-scheme`로 시스템 설정을 따랐는데, 사용자가 직접 on/off 하고 싶어 함. → `html[data-theme]` 속성으로 색 변수를 덮어써서(속성 선택자가 `:root`보다 우선) **수동 선택이 시스템 설정을 이기게** 함. 선택은 localStorage 저장, `<head>`에 초기 적용 스크립트를 넣어 로드 시 **깜빡임 방지**.
- **기록 수정**: 내역 항목을 누르면 그 값(구분·날짜·금액·메모·분류)이 폼에 채워지고, 저장하면 새로 추가하지 않고 **그 기록을 제자리 수정**. `editingId` 상태로 추가/수정을 분기. 삭제(✕)는 `stopPropagation`으로 수정 클릭과 충돌 방지, 수정 중이던 항목을 지우면 폼도 초기화.

### 9. 폰↔PC 자동 동기화 (Firebase 도입)

- localStorage는 **기기마다 따로**라 폰과 맥북이 다르게 보이는 게 근본 한계. 사용자가 이를 정확히 짚어서, 구글 로그인 + Firestore로 **자동 동기화**를 붙임(혼자 사용, 무료 Spark 플랜).
- **비침습적 설계**: 기존 IIFE는 그대로 두고 `window.LedgerApp`(getData/setData/toast) 훅만 노출. Firebase는 별도 `<script type="module">`에서 `window.CloudSync`를 등록하고, 앱의 `save()`가 로그인 상태면 클라우드에도 push. → 로컬 전용 동작을 하나도 깨지 않고 동기화를 얹음.
- **미설정 시 우아하게 실패**: `firebaseConfig`가 placeholder면 SDK를 초기화하지 않고 버튼을 "설정 필요"로 비활성화. 앱은 로컬로 정상 동작 → 설정 전이라도 라이브 사이트가 안 깨짐.
- **데이터 모델**: 기록 전체를 문서 하나(`ledgers/{uid}.entries`)에 저장 — 기존 "배열 통째로 save" 패턴과 그대로 맞아 코드가 단순. 단, 문서 1MB 제한(대략 수천 건)이 있어 아주 오래 쓰면 문서-per-기록 구조로 이전 필요.
- **병합/삭제 함정**: 매 시작마다 로컬∪클라우드로 합치면 **삭제한 기록이 되살아남**. 그래서 "클라우드가 진실"로 두고, 로컬에만 있는 기록이 있을 때만 합칠지 물어보게 함.
- **CDN 버전 확인**: `gstatic` Firebase SDK URL은 버전이 틀리면 404 → 커밋 전 `curl`로 `10.14.1` 존재를 확인.

### 10. iPhone용 PWA로 만들기

- 아이폰에서 앱처럼 쓰려고 PWA화. iOS는 **홈 화면 아이콘을 manifest가 아니라 `apple-touch-icon`(PNG)** 에서 읽고, 전체화면은 `apple-mobile-web-app-capable` 등 **애플 전용 meta**로 켜짐 → 그에 맞춰 태그 구성.
- 아이콘은 Pillow로 인디고 그라데이션 + ₩ 심볼 PNG(512/192/180) 생성.
- 서비스 워커는 **네트워크 우선 → 실패 시 캐시**(오프라인 실행). 단, `origin` 검사로 **우리 사이트 파일만** 가로채고 Firebase/gstatic 등 외부 요청은 건드리지 않음(로그인·동기화 안 깨지게).
- 겪은 함정: 사용자가 로컬 `file://`로 앱을 열어 기록이 그 origin에만 갇혀 링크(https)에선 안 보였음. **localStorage는 origin별로 분리** → 항상 배포 링크(https)만 쓰도록 안내. 복구는 JSON 백업/복원으로.

### 11. 아이폰에서 날짜칸이 오른쪽으로 삐져나옴

- 폰에서 보니 날짜 입력칸이 다른 칸과 정렬이 안 맞고 화면 오른쪽으로 넘침. **iOS Safari의 `<input type="date">`는 고유(intrinsic) 너비가 크고 줄어들지 않으며**, grid 아이템의 기본값(`min-width:auto`)과 겹쳐 `1fr` 트랙이 컨테이너보다 넓어진 것. 데스크톱 크로미움에선 재현되지 않는 iOS 전용 증상.
- 해결: 입력칸 공통 `min-width:0` + 날짜칸에 `-webkit-appearance:none`, `max-width:100%`. iOS가 날짜 텍스트를 가운데 정렬하는 것도 `::-webkit-date-and-time-value{text-align:left}`로 다른 칸과 맞춤.

### 12. 실수 삭제 → 확인창 → 휴지통

- ✕ 한 번에 즉시 삭제되는 구조라 수입 기록을 실수로 날림. 1차로 **confirm 확인창**(날짜·금액·분류·메모 표시)을 추가.
- 2차로 **휴지통** 도입: 삭제하면 지우는 대신 `trash` 배열로 이동. `#trash` 해시 라우트의 별도 페이지에서 **개별/일괄(체크박스+전체 선택) 복구·영구 삭제**. 해시 라우팅이라 iOS 뒤로가기 스와이프도 자연스럽게 동작.
- 휴지통도 Firestore 문서에 `trash` 필드로 **동기화**(로그인 시 id 기준 합집합 병합). 옛 버전 클라이언트가 `trash` 필드 없이 문서를 덮어써도 로컬 휴지통을 지우지 않도록 `Array.isArray` 체크 후에만 반영.
- 영구 삭제만 "되돌릴 수 없습니다" confirm을 거침. 복구는 되돌릴 수 있으니 확인 없이 바로.

### 13. "고쳤는데 폰에서 안 바뀌어요" — 배포 경로·캐시 2중 함정

- 함정 1: 작업 브랜치에만 푸시하고 **main에 안 합침**. GitHub Pages는 main에서 배포하므로 폰에서 아무리 새로고침해도 반영 안 됨. → main 병합 후 해결.
- 함정 2: main에 합쳤는데도 안 보임. **GitHub Pages는 `Cache-Control: max-age=600`(10분)으로 응답**하고, 서비스 워커의 "네트워크 우선" `fetch()`도 이 **HTTP 캐시를 그대로 탐**. Actions 배포 기록으론 성공인데 폰은 10분 전 버전을 보고 있었음.
- 해결: SW에서 `fetch(new Request(url, {cache:"no-cache"}))`로 **항상 서버에 재검증**(ETag 덕에 비용 거의 없음) + 화면 하단에 **버전 표시(v4, v5…)**를 넣어 지금 어떤 빌드인지 눈으로 확인 가능하게. PWA 갱신은 "완전 종료 → 재실행"을 두 번(첫 실행에서 새 SW 설치, 두 번째부터 적용).

### 14. 터치 기기의 hover 잔상 두 가지

- ✕를 눌렀다 확인창에서 취소하면 **빨간 눌림 표시가 계속 남음**. iOS는 탭을 hover로 취급하고 **다른 곳을 탭할 때까지 hover 상태를 유지**하기 때문. → 모든 `:hover` 규칙을 `@media (hover:hover)`로 감싸 **마우스 기기에서만** 적용. 폰에선 잔상 없음, PC에선 기존과 동일.
- 수정하다 취소해도 항목 강조가 남는 건 별개의 진짜 버그: 취소 버튼이 폼만 리셋하고 **목록을 다시 그리지 않았음**. → 취소 시 `renderList()`도 호출.

### 15. N빵 정산 — 지출·수입 겸용 분류와 "순지출" 통계

- 2명이 20,000원을 내가 결제하고 10,000원을 돌려받는 경우: 지출 20,000 + 수입 10,000을 각각 기록하되, 식비 통계엔 **차액 10,000만** 잡히길 원함.
- 식비·카페/간식을 **수입 분류에도 추가**하고, 지출·수입 양쪽에 존재하는 분류(`NET_CATS` — 식비, 카페/간식, 쏘)는 그 분류의 수입을 **분류별 지출 통계와 월별 추이 차트에서 차감**해 순지출 기준으로 표시.
- 상단 수입/지출/잔액 카드는 기록 그대로(실제 돈 흐름) 유지 — 통계만 순지출. 정산이 지출과 다른 달에 기록되면 음수가 될 수 있어 **0으로 클램프**.

### 배운 점 요약

- 데이터가 브라우저에만 있는 구조는 **백업 기능이 필수**.
- 비공개 저장소는 무료 Pages가 안 된다 — 웹 배포 계획이면 처음부터 공개로.
- 로컬 HTML 자동 테스트는 `file://` 대신 **간이 HTTP 서버**로.
- 입력값이 "없는" 건지 "안 보이는(잘린)" 건지 구분할 것 — 폭 문제일 때가 많다.
- `type="number"`는 콤마 표시가 안 됨 — 포맷이 필요하면 `type="text"`+`inputmode="numeric"`.
- 수동 테마는 `html[data-theme]` 속성 + localStorage, `<head>` 초기 스크립트로 깜빡임 방지.
- 리스트 항목에 클릭 액션을 여러 개 얹을 땐(수정 vs 삭제) `stopPropagation`으로 분리.
- 기존 코드에 외부 연동을 얹을 땐 얇은 훅(`window.LedgerApp`)만 노출하고 연동부를 분리 — 기존 동작을 안 깨뜨림.
- 클라우드 동기화에서 "배열 통째로 병합"은 삭제 되살아남 문제가 있음 → 진실의 원천을 한쪽으로 정하기.
- Firebase 웹 config(apiKey 등)는 비밀이 아님. 보안은 Firestore 규칙으로.
- iOS의 `<input type="date">`는 고유 너비가 크고 텍스트가 가운데 정렬 — `appearance:none` + `min-width:0` + `::-webkit-date-and-time-value`로 정리.
- 터치 기기에선 탭이 hover로 남는다 — `:hover` 스타일은 `@media (hover:hover)` 안에만.
- "배포했는데 안 보인다"는 코드보다 **경로와 캐시부터**: 브랜치가 배포 브랜치에 합쳐졌는지 → Pages HTTP 캐시(10분) → SW/PWA 캐시 순으로 의심.
- 파괴적 동작은 확인창보다 **되돌릴 수 있는 구조(휴지통)**가 한 수 위. 확인창은 습관적으로 눌러서 뚫린다.
- 화면 어딘가에 버전 표시를 넣어두면 "지금 어떤 빌드를 보고 있나" 논쟁이 사라진다.
- 정산(N빵)은 수입·지출을 상계한 "순지출" 통계가 필요 — 단, 원장(카드 요약)은 기록 그대로, 통계만 상계.

### 16. 모바일 화면 개편 — 긴 한 페이지에서 탭 구조로

- 기능이 늘면서 동기화·요약·입력·내역·통계·백업이 한 화면에 세로로 길게 쌓여, 휴대폰에서 원하는 기능을 찾기까지 스크롤이 길어짐.
- `홈 / 내역 / 추가 / 통계 / 설정` 5개 하단 탭을 추가하고, 각 화면을 해시 라우트(`#home`, `#records` 등)로 나눔. 해시 라우팅은 iOS 뒤로가기 제스처와 새로고침 후 현재 화면 유지에 유리함.
- 기록 입력은 별도 `추가` 화면으로 이동시키고, 홈에는 기간 요약·최근 내역·간단 통계만 남겨 첫 화면의 정보 밀도를 낮춤.
- 날짜 이동에 `오늘` 버튼을 추가해 과거 기간을 살펴본 뒤 현재 기간으로 한 번에 돌아오게 함.

### 17. 길게 누르기 메뉴와 실행 취소

- 기존 내역은 짧게 누르면 바로 수정으로 들어가고, 삭제 버튼은 화면에 항상 노출되는 구조였음. 모바일에서 실수 삭제와 수정 진입 혼동 가능성이 있었음.
- 내역을 길게 누르거나 키보드의 컨텍스트 메뉴 키를 누르면 액션 시트를 열어 `복제 / 수정 / 휴지통 이동`을 선택하도록 변경함. 마우스 환경에서는 `⋯` 메뉴 버튼도 제공함.
- 삭제·복제·수정은 직전 상태 스냅샷을 저장해 5초 동안 `실행 취소`할 수 있게 함. 휴지통과 기존 복구 기능은 그대로 유지함.
- 길게 누르기 타이머는 이동 중 취소하고, 지원 기기에서는 짧은 진동으로 메뉴가 열렸음을 알림. `prefers-reduced-motion` 사용자에게는 불필요한 애니메이션을 줄임.

### 18. 접근성 보강

- 입력 필드에 실제 `<label>`을 붙이고, 수입/지출·분류·기간 선택 상태는 `aria-pressed`로 노출함.
- 아이콘 버튼에 `aria-label`, 기간·동기화 상태에 `aria-live`, 통계 SVG에 설명 역할을 추가함.
- 내역 항목은 키보드로 포커스한 뒤 Enter/Space로 수정할 수 있고, Shift+F10 또는 ContextMenu 키로 액션 시트를 열 수 있음.
- 액션 시트는 포커스를 내부 버튼에 두고 Tab 순환과 Escape 닫기를 지원함. 토스트는 `role="status"`로 화면 읽기 프로그램에도 전달함.

### 19. 이번 개편에서 생긴 시행착오

- 처음에는 기존 `#stats`, `#trash` 라우트와 새 탭 라우트를 섞어 화면을 숨겼기 때문에, 특정 화면에서 상단 기간 선택기가 남거나 제목이 잘못 보일 수 있었음. 화면별 표시 요소와 제목을 `route()`에서 함께 갱신하도록 정리함.
- 기록 항목에 클릭 수정과 길게 누르기를 동시에 넣으면 긴 누른 뒤 클릭 이벤트가 한 번 더 발생함. `longPressed` 상태로 후속 클릭을 무시하고, 메뉴 버튼은 이벤트 전파를 중단함.
- 토스트에 실행 취소 버튼을 추가하면서 기존 메시지 전용 구조와 충돌함. 메시지·액션 버튼을 분리하고, 액션이 있을 때만 타임아웃을 5초로 늘림.
- 모바일 하단 탭이 콘텐츠를 가리는 문제를 피하려고 본문 하단 여백과 iOS safe-area inset을 함께 적용함.
- `type="button"`을 지정하지 않은 폼 버튼은 Enter 입력 시 의도하지 않은 제출을 일으킬 수 있어, 입력 폼의 버튼 역할을 명시적으로 분리함.
- 실제 모바일 브라우저에서 확인할 때는 서비스 워커가 이전 HTML을 계속 제공할 수 있음. 캐시 버전을 올리고, 앱을 완전히 종료한 뒤 다시 실행해야 새 서비스 워커가 적용됨.

### 20. GitHub에 푸시했는데 Firebase 앱은 그대로였던 문제

- 휴대폰 홈 화면 앱은 GitHub Pages가 아니라 Firebase Hosting URL에서 설치했는데, 코드를 `main`에 푸시하기만 하고 Firebase에는 별도로 배포하지 않아 새 버전이 나타나지 않았음.
- Git 저장소 푸시와 Firebase Hosting 배포는 서로 다른 과정이다. Firebase 앱을 갱신하려면 `firebase deploy --only hosting`까지 실행해야 함.
- 배포 후에도 Firebase 기본 응답이 HTML·서비스 워커에 `Cache-Control: max-age=3600`을 주고 있어 iOS 홈 화면 앱이 이전 파일을 최대 한 시간 더 사용할 수 있었음.
- `firebase.json`에서 `/`, `/index.html`, `/sw.js`를 `no-cache, no-store, must-revalidate`로 설정하고, 서비스 워커 등록에 `updateViaCache: "none"`과 `registration.update()`를 추가해 앱을 열 때 최신 버전을 즉시 확인하도록 보강함.
